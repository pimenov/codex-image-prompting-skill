# Example: UI / system diagram

User request:

```text
Prepare an image prompt for a dashboard mockup showing how a team reviews incoming research notes.
```

Polished prompt:

```text
Create a production-quality 16:10 desktop dashboard mockup for a fictional product named "Signal Desk".

Product context:
A research team uses this screen to review incoming notes, group them by topic, and decide which items need owner review.

Screen structure:
- Header: include "Signal Desk", "Review queue", and a search field.
- Left sidebar: navigation labels "Inbox", "Topics", "Owners", "Decisions".
- Primary panel: a table of research notes with columns "Source", "Topic", "Confidence", "Owner", "Next action".
- Secondary panel: a compact knowledge graph preview and a status card labeled "Needs review".

Data and copy:
Use realistic values: "Customer calls", "Product feedback", "High", "Assign owner", "Merge duplicate".

Visual system:
Quiet SaaS interface, dense but readable layout, soft gray background, white panels, blue and green status accents, precise icon alignment.

Quality constraints:
Crisp typography, real product density, no lorem ipsum, no broken icons, no random chart labels, no fake brand marks.
```
