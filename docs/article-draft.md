# Skill для визуальных промптов: как Codex помогает делать картинки осмысленно

Большая часть разговоров про генерацию изображений быстро уходит в модели,
API, ключи, параметры и стоимость. Но в ежедневной работе часто первым
ограничением оказывается не модель, а промпт: мы просим «сделай красиво», а
получаем картинку без ясного формата, с плохой типографикой, случайными
надписями и непонятной композицией.

Этот пакет появился из простого наблюдения: для Codex Desktop полезен отдельный
prompt-first skill, который сначала проектирует визуальную задачу, а затем
сразу запускает native image generation, если пользователь просит картинку.
Его задача — помочь агенту пройти полный цикл: идея, промпт, изображение,
оценка результата и следующая итерация.

## Что делает skill

`codex-image-prompting` превращает сырой запрос в структурированный визуальный
бриф, а затем может использовать этот бриф для генерации:

- какой это артефакт: постер, карточка, схема, UI mockup, карусель;
- какой формат и layout нужны;
- какой текст должен быть видимым и точным;
- где должны быть safe zones;
- что модель должна сохранить при редактировании;
- какие ошибки нужно явно предотвратить.

Вместо «нарисуй современный постер» skill ведет Codex к промпту вида:

```text
Square 1:1 social card for a professional workshop about practical AI workflows.
Promotional hierarchy:
1. Main title largest: "Practical AI workflows"
2. Supporting line: "From messy requests to repeatable systems"
...
```

## Что я взял из GPT-Image2-Skill

Хорошая идея в `wuyoscar/GPT-Image2-Skill` не только в генерации через API.
Самое ценное там — prompt craft:

- начинать с формата и layout;
- держать exact visible text в кавычках;
- описывать UI как product spec;
- описывать диаграммы через зоны, стрелки, легенды и смысл цветов;
- для edit prompts разделять `Preserve` и `Change`;
- для сложных сцен использовать структурированные блоки.

В этом пакете я оставил именно эту часть и убрал API/CLI слой. Получился skill,
который можно использовать внутри Codex Desktop как "дизайн-ворота" перед
генерацией и как удобный цикл для итераций по готовому изображению.

## Почему prompt-first лучше, чем просто картинка

Если нужно быстро проверить визуальное направление, Codex Desktop image
generation уже дает хороший быстрый контур. Но skill добавляет дисциплину:
он не дает агенту сразу бросаться в картинку, пока не названы формат, текст,
иерархия, ограничения и вероятные ошибки.

При этом пользователь не остается с одним текстовым промптом на руках. Если он
просит изображение, Codex должен сразу использовать native image generation,
показать результат и дать короткую оценку: что получилось, что сломалось, что
лучше поправить во втором проходе.

Это особенно полезно для:

- обложек и social cards;
- схем и инфографики;
- UI mockups;
- визуальных объяснений;
- каруселей для соцсетей.

## Урок каруселей

Самая интересная проверка была с seamless carousel: одна широкая картинка,
которая режется на несколько квадратных слайдов.

Тут быстро выясняется важное ограничение: image generation может понять идею
панорамы, но не обязана вернуть точный production-холст. Если для пяти слайдов
`1080x1080` нужен master `5400x1080`, а генератор вернул маленькую панораму
вроде `2172x724`, это не финальный материал. Если ее нарезать и растянуть,
детали не появятся - мы просто увеличим маленькие фрагменты.

Поэтому skill разделяет три режима:

- `concept` — быстро получить визуальное направление;
- `production` — собрать детерминированный холст нужного размера и наложить типографику отдельно;
- `overlap experiment` — пробовать последовательные части с нахлестом, но не считать это пиксельно надежным.

Главный вывод: AI хорошо помогает с visual direction, но публикационная
геометрия должна быть детерминированной.

## Как повторить workflow

1. Установить skill в Codex.
2. Попросить: `Use $codex-image-prompting: generate ...`
3. Codex готовит промпт, запускает native image generation и показывает результат.
4. Проверить, что результат соответствует формату, layout, exact text, style и avoid-line.
5. Для карусели выбрать режим: concept или production.
6. Для production проверить размер master image перед нарезкой.

Проверка геометрии:

```bash
python3 scripts/check_carousel_geometry.py --image path/to/master.png --slides 5 --export-size 1080
```

Если результат `concept-only`, не выдавайте его за финальную карусель.

## Почему это полезно людям

Такой skill не обещает магии. Он делает другое: превращает визуальные запросы в
повторяемый процесс. Это помогает агенту работать не как случайный генератор
картинок, а как внимательный помощник, который сначала формулирует задачу,
затем выбирает правильный визуальный контракт, генерирует изображение и помогает
понять, что именно стоит улучшить в следующей итерации.
